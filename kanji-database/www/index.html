概要

『漢字データベース計画』は、Unicode / UCS(Universal Coded Sets) によって符号化された漢字（CJK統合漢字）の情報交換・検索照合・分析に役立つ様々なデータベースを整備することを目的に、2003年度に日本学術振興会科学研究費補助金・研究成果公開促進費（データベース）の援助を受けて開始しました。

各種漢字データファイルは、CVSにて管理されており、CVSサーバから取得することができます。IDS (Ideographic Description Sequence) については、ISO/IEC 10646:2003 箇条 F.3 を参照してください。

字形ＩＤＳデータ

字形ＩＤＳデータは、UCS漢字の字形を可能な限り、IDSで表記する試みです。本データは、CHISE 漢字情報構造データのブランチ（枝）として、IRGの標準化作業の一環として整備しています。

CHISE IDSの主ブランチと比較して、本ブランチは (1) 部品の表現にUCSのAnnex Sの規定を適用する (2) CJK統合漢字の字形差をできるだけ明確にする (3) DC (Description Character) にはUCS文字を可能な限り使用し、やむを得ない場合に限って、CDP外字を使用するようにしています。

本データは ISO/IEC JTC1/SC2/WG2/IRG における、拡張C・D漢字集合の標準化作業に利用されています。またその際、各国・各地域のメンバーからデータの間違いなどを指摘して頂きました。拡張Ｃ文字集合の候補漢字は、本データを参考に各国・各地域の担当者の手によりＩＤＳ化されました。このデータを利用することで、拡張Ｃ文字集合の品質の向上を図っています。

  • 字形ＩＤＳデータ（CVSサーバ・ブランチ名：kawabata）

解字ＩＤＳデータ

解字ＩＤＳデータは、UCS漢字を、字の成り立ちからIDS化する作業を行っています。各漢字の成り立ちについては、説文解字注による記述を中心に、字通、漢語林、オンライン説文（新字源）よる解釈を参考にしています。説文解字注の記述を重視しているため、必ずしも現在通用している説と同じとは限りません。各カラムはタブで区切られており、以下の意味を持ちます。

カラム書式説明
1U+xxxxxUCS符号
2char文字
3→charこの漢字が、charから派生した異体字であることを示します。
3IDSこの漢字が会意・形声文字の場合はIDSを示します。指事・象形文字の場合は文字そのものを示します。
4六書この文字が形声文字の場合は、“Ｘ聲”として示します。会意形声文字の場合は、“Ｘ亦聲”と表記する場合があります。
5説文解字注番号説文での位置を、“《Ｒxxx》”で表記します。Ｒは部首、xxxは部首での番号を示します。
6コメント他の出典や解字の異説などを、“【・・・】”で示す場合があります。

解字ＩＤＳデータは、象形文字・会意文字を康煕字典部首画数の順に並べ、その異体字や形声文字をそれぞれの後ろに配列しています。

  • 解字ＩＤＳデータ（テキストファイル）

ＵＣＳ漢字の統合・包摂規準

「ＵＣＳ漢字の統合・包摂規準」は、JIS X 0208/0213で規定されている包摂規準に類似するものを、UCS漢字に対して作成しようとする試みです。JISの連番をベースに作成し、UCS漢字の入力・検索の参考情報として示すことを目的とします。

說文解字注データ

説文解字注データは、段玉裁の「説文解字注」をUCSで符号化することを目指します。2007年現在、まだ３割程度の親字が未入力です。本データは、日本学術振興会科学研究費・基盤研究C「次世代古典文献データベース構築の基礎的研究」（平成14〜16年度、課題番号：14510494、研究代表者：村越貴代美）による成果の一部を利用させて頂いています。

説文解字注データは、經韵樓臧版（上海古籍出版社）影印本を 特定の XML スキーマに従うように再構成しています。これにより、XMLデータ言語（XQuery等）でデータ処理ができることを目指します。

宋本廣韻データ

宋本広韻データは、周祖謨「校正宋本廣韻」をUCSで符号化することを目指します。本データは、日本学術振興会科学研究費・基盤研究C「次世代古典文献データベース構築の基礎的研究」（平成14〜16年度、課題番号：14510494、研究代表者：村越貴代美）による成果の一部を利用させて頂いています。

宋本廣韻データは、指定された XML スキーマに基いて入力されてり、XML処理系でデータ処理ができることを目指します。反切部分はマークアップで示され、そこから推定される中古音の例をIPAで示します。親字ＩＤは工藤祐嗣氏の「巻数(1-5)・丁数(2桁数)・表裏面(a/b)・行数・番号」を利用させて頂いています（→参考論文）。

篆書体フォント “SW.ttf” UCS対応表

本データは、説文解字検索データベースや、「《说文解字》全文检索」(ISBN 7-80652-327-8)などの書籍で入手可能な篆書体 フォント “SW.ttf”と、UCS符号との対応を示します。本データはまだ未完成です。SW.ttf フォントは、説文解字（大徐本）に基づいています。それぞれのカラムの意味は以下の通りです。

カラム番号説明
1SW.ttfの文字コード
2説文解字の整理番号（仮番号）
3対応する文字（複数ある場合は、小篆に近い順番）

なお、本フォントをインストールすると、HTMLのスタイルで "font-family: SW;" を指定することで、対応する小篆をブラウザで表示することができます。

異体字データベース

異体字データベースは、各種文献に記載されている異体字情報を収集・整理したものです。これらは情報処理学会 試行標準 IPSJ-TS 0008:2007 「大規模漢字集合の異体字構造」に記載されている異体字情報と、基本的には同一のものです。なお、UCSでは包摂されるか、表現できないものは U+0FFFD で記載されています。

variants-jisx0213.txt JIS X 0213に記載されている異体字 (JIS X 0208 同士を含む)
variants-ucs-scs.txt ISO/IEC 10646 Annex S.3 例字（原規格分離一覧表）

replaceables-douon.txt 上記に基づく書き換え一覧。# asahi とコメントがあるものは、朝日字体によるもの、# context-dependent とコメントがあるものは、熟語に依存するもの。
variants-joyo.txt 常用漢字表に掲出する新旧字
variants-jinmei2-1.txt 人名漢字表 別表２の１で同一の字種とされる漢字
variants-jinmei2-2.txt 人名漢字表 別表２の２で規定される異体字

ＵＣＳ類似漢字一覧表

ＵＣＳ重複漢字一覧表

重複漢字一覧表は、ＵＣＳ統合漢字において、同一字形で大漢和辞典・康煕字典では「音・義」の違いが見られない文字が、異なる符号位置に重複して符号化されているものの一覧表です。 同じ字形かどうかは、漢字の字形の差異がUCS漢字の統合・包摂規準に該当しないほど些細な違い（止め・はねや、線の相対的な長さの違い等）しかないかどうかで判断します。

漢字フォント

漢字データプロジェクトでは、フリーなフォントの整備を行っています。成果となるフォントは『花園明朝』と名付けられ、別サイトにて公開されています。現在、第二水準までの計6,356文字が用意されており、字形編集ウィキであるGlyphWikiにより誰もが簡単に拡張・編集することができるようになっています。なお、本フォントは漢字のみで構成され、「ひらがな」等はありませんのでご注意ください。

Unicode

Unicodeは、Unicodeコンソーシアムと呼ばれる民間の団体が定めた私的な文字コード規格です。ISO/IEC 10646と文字表が一致するように、規格を定めています。

  • Unihanデータベースは、Unicode漢字の部首・画数・意味・発音・異体字などの様々なデータを整備・公開しています。
  • “UTR #37: Ideographic Variation Database”は、Unicodeコンソーシアムに漢字の異体字を登録する手続きを定めています。

国際漢字コード規格関連組織

ISO/IEC JTC1/SC2/WG2/IRG

ISO/IEC JTC1/SC2/WG2/IRGは、国際16bit文字コード標準化団体であるWG2の付託を受けて、漢字集合の策定などの作業を行っています。

  • IRG N 1381は、拡張漢字B集合のマルチカラムチャートの最新ドラフトです。拡張漢字B集合は2002年に制定されましたが、例示字形は１書体しかありませんでした。今後、この表は６欄併記に改められる予定です。
  • IRG N 1351は、拡張漢字C集合の最新ドラフトです。拡張漢字C集合はほぼfixされており、特に問題がなければ、ISO/IEC 10646:2003 Amendment 5 で制定される予定です。
  • IRG N 1353は、拡張漢字D集合の最新ドラフトです。拡張漢字D集合は現在、拡張漢字Cに続く新しい漢字集合として審議中です。
  • IRG N 1380は、『緊急に必要な文字集合』の最新ドラフトです。『緊急に必要な文字集合』は、拡張漢字D集合の候補文字が締め切られた後に、各国で必要になった文字を集めたものです。
